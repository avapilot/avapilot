Get started in 3 minutes

01

Install

You need Python 3.10+ and uv.

git clone https://github.com/avapilot/avapilot.git
cd avapilot
uv pip install -e .
avapilot seed
02

Connect to your AI

Pick your client. Copy-paste. Restart the app.

# Claude Desktop → claude_desktop_config.json
{
  "mcpServers": {
    "avapilot": {
      "command": "avapilot",
      "args": ["gateway", "--mode", "trade", "--chain", "fuji"]
    }
  }
}
# OpenClaw / Cursor / Windsurf — same pattern
command: avapilot
args: gateway --mode trade --chain fuji

Start with --chain fuji (free testnet). Switch to --chain avalanche for real money.

03

Talk to Avalanche

"What DeFi protocols are available?"

What you can do

Swap & trade

Trader Joe, Pangolin, Pharaoh — get quotes, execute swaps, add liquidity.

Lend & borrow

Aave V3, Benqi — check rates, supply collateral, borrow.

Liquid staking

sAVAX, ggAVAX — stake AVAX, check yields.

Create L1 blockchains

Subnets, validators, staking — one chat message.

Read any contract

Call any function on any verified contract on Avalanche.

Cross-chain

C-Chain ↔ P-Chain transfers. Bridge via Stargate.

Safety modes

# Read only — can't spend money
avapilot gateway --mode read --chain avalanche

# Trading — send, swap, wrap, stake, create L1s
avapilot gateway --mode trade --chain fuji

# Full — trade + deploy smart contracts
avapilot gateway --mode full --chain fuji

Your key never leaves your machine. All signing is local. We're open source — verify it yourself.

Get your protocol in front of every AI agent

One PR. Every AI agent connected to AvaPilot discovers your contracts. No SDK. No integration work. No maintenance.

01

Check what AI sees from your contract

avapilot scan 0xYourContractAddress

Shows every callable function, inputs, outputs, read vs write. This is exactly what AI agents will see.

02

Register

Use our form — it generates everything for you.

Registration form →

Or add to avapilot/registry/seed.py manually:

{
    "name": "Your Protocol",
    "contracts": [
        {"address": "0xRouter...", "label": "router"},
        {"address": "0xFactory...", "label": "factory"},
        {"address": "0xStaking...", "label": "staking"},
    ],
    "description": "One line description",
    "category": "DeFi",
    "website": "https://yourprotocol.com",
}

Multi-contract supported. Add router, factory, vaults — whatever. Each gets its ABI auto-fetched and proxy-resolved.

How agents discover your protocol

Agent: "What can I do on Avalanche?"
→ search_services()                        # 18 protocols

Agent: "I want to swap on Trader Joe"
→ service_functions("Trader Joe")          # 60 functions, 3 contracts

Agent: "Swap 1 AVAX for USDC"
→ call_service("Trader Joe", "getAmountsOut", ...)     # quote
→ send_service_tx("Trader Joe", "swapExact...", ...)   # execute

What you get

Auto ABI fetch

We pull ABIs from Snowtrace. Detect proxies. You just provide addresses.

Multi-contract

Router + factory + vaults — all under one service name.

Every MCP client

Claude, Cursor, OpenClaw, Windsurf — your protocol works everywhere.

Zero maintenance

Once merged, it works forever. No SDK. No integration calls.
